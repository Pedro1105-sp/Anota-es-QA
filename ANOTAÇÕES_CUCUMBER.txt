*** CUCUMBER ***

BDD -> DESENVOLVIMENTO ORIENTADO A COMPORTAMENTO
	BDD É UM DESENVOLVIMENTO ORIENTADO AO COMPORTAMENTO
	BDD NÃO É "DADO, QUANDO e ENTÃO"
		DADO, QUANDO e ENTÃO, É UMA SINTAXE GHERKIN

TDD -> DESENVOLVIMENTO ORIENTADO A TESTES
	
SE A APLICAÇÃO JÁ ESTÁ PRONTA, ENTÃO NÃO É BDD, POIS BDD É DESENVOLVIMENTO ORIENTADO AO COMPORTAMENTO

DDT -> TESTE ORIENTADO A DADOS

CUCUMBER VINCULAR O GHERKIN COM O TESTE AUTOMATIZADO



RUBY -> @VARIAVEL, tem que acrescentar o @ para ser uma váriavel global

EXECUTAR OS TESTES E PEGAR JÁ OS LOGS "bundle exec cucumber -f html -o logs/report.html" OU PARA APARECER NO CONSOLE
	"bundle exec cucumber -f pretty -f html -o logs/report.html"
