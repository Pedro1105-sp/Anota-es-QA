CONSIDERAÇÕES ACERCA DA AUTOMAÇÃO

WHAT (O QUE?)  - "O QUE SERÁ AUTOMATIZADO"
	- SABER QUAIS SISTEMAS E RESPECTIVOS CENÁRIOS DE TESTES DEVERÃO SER AUTOMATIZADO
	- QUAIS SISTEMAS
	- QUAIS MODOS
	- QUAIS FUNCIONALIDADES
		- AS CRITICAS
		- TODAS AS FUNCIONALIDADES

WHO (QUEM?)  -  QUEM DA EQUIPE SERÁ ENCARREGADO DE REALIZAR ESSA ATIVIDADE DE AUTOMAÇÃO
	- QUAIS DOS INTEGRANTES É MAIS ADEQUADO
	
	

WHERE(ONDE?)  -  SABER ONDE ESSE PROCESSO SERÁ IMPLEMENTADO

WHEN (QUANDO?)  -  ESSA ATIVIDADE DE AUTOMAÇÃO É URGENTE
	- MOMENTO CERTO PARA COMEÇAR
	
WHY (POR QUE?)  -  POR QUE A EMPRESA QUER AUTOMATIZAR OS TESTES
	- MOTIVO NECESSÁRIO PARA IMPLEMENTAR OS TESTES AUTOMATIZADOS
	

HOW (COMO?)  -  COMO SERÁ IMPLEMENTADO ESSA AUTOMAÇÃO
	- QUAIS SÃO AS TÉCNICAS 
	- VAI SER IMPLEMENTA ATRAVES UM TESTE DE REGRESSÃO, DE PERFORMACE....
	- QUAIS FERRAMENTAS SERÃO UTILIZADAS, UMA FERRAMENTA PAGA, UMA COMPLEXA, PRECISA DE TREINAMENTO
	
HOW MUCH (QUANTO CUSTA?)  -  VAI PRECISAR CONTRATAR MAIS PROFISSIONAIS, QUAIS SAO OS CUSTOS, CAPACIDADE, FERRAMENTAS...


**** PRINCIPIOS DA AUTOMAÇÃO DE TESTE ****
- PROJETAR OS CASOS DE TESTES PARA DEPOIS AUTOMATIZAR
- NÃO AUTOMATIZAR 100% DE TUDO, FOCAR NA PRIORIDADE 
- PROJETAR A AUTOMAÇÃO DE FORMA QUE SEJA FÁCIL DE REVISAR
- INICIAR AUTOMAÇÃO DE TESTE MAIS CEDO POSSÍVEL EM UM PROJETO


**** O QUE É SELENIUM ****
- É UMA SUÍTE COMPORSTA POR 4 FERRAMENTAS DE APLICAÇÕES DE TESTES PARA WEB 
	- IDE, REMOTE CONTROL, WEBDRIVER e GRID
	



***** PRATICA SELENIUM ****
- COMANDOS DE VALIDAÇÃO DE LAYOUT
	- ASSERT:
		COMANDO PARA VERIFICAR SE EXISTE UM VALOR NAQUELE ELEMENTO
	- STORE:
		ARMAZENA UM VALOR DAQUELE ELEMENTO EM UMA VÁRIAVEL
	- ASSERT ELEMENT PRESENT:
		ELE VERIFICAR SE O ELEMENTO ESTÁ PRESENTE
